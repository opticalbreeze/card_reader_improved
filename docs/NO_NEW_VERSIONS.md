# バージョン管理方針

## ⚠️ 重要: 新しいバージョンを作成しない

**このプロジェクトでは、新しいバージョンのクライアントファイルを作成しないでください。**

## 現在のファイル構成

### ラズパイ版
- **`pi_client.py`** - 唯一のメインファイル（これのみ使用）

### Windows版
- **`win_client.py`** - 唯一のメインファイル（これのみ使用）

## 起動スクリプトの参照先

すべての起動スクリプトは以下のファイルを参照しています：

- `start_pi_simple.sh` → `pi_client.py`
- `start_pi.sh` → `pi_client.py`
- `config.py` → `pi_client.py`（ラズパイの場合）

**これらの参照先を変更しないでください。**

## 理由

1. **ラズパイ側でバッチファイル（シェルスクリプト）で起動している**
   - 参照先が変わると、何が悪いのかわからなくなる
   - 起動スクリプトを修正する必要が生じる

2. **混乱を避けるため**
   - 複数のバージョンがあると、どれを使えばいいかわからなくなる
   - メンテナンスが複雑になる

3. **シンプルさを保つため**
   - 1つのファイルに機能を集約することで、管理が容易になる

## 既存のバックアップファイル

以下のファイルはバックアップとして存在しますが、**使用しません**：

- `pi_client_simple.py` - バックアップ（使用しない）

## 変更が必要な場合

機能を追加・変更する場合は、**既存のファイルを直接編集**してください：

- ラズパイ版: `pi_client.py`を編集
- Windows版: `win_client.py`を編集

新しいファイルを作成するのではなく、既存のファイルを更新してください。
